# 多智能体专业团队（独立创始人设置）

独立创始人身兼数职——战略、开发、营销、销售、运营。在这些角色之间切换上下文会破坏深度工作。招聘既昂贵又缓慢。如果你能组建一个小型、专业的 AI 智能体团队，每个智能体都有独特的角色和个性，并且都可以从单个聊天界面控制，那会怎样？

此用例将多个 OpenClaw 智能体设置为一个协调团队，每个智能体专注于一个领域，通过共享记忆进行交流，并通​​过 Telegram 进行控制。

## 痛点

- **一个智能体无法做好所有事情**：当兼顾战略、代码、营销研究和业务分析时，单个智能体的上下文窗口很快就会填满
- **没有专业化**：通用提示产生通用输出——编码智能体不应该同时也撰写营销文案
- **独立创始人倦怠**：你需要一个团队，而不是另一个需要管理的工具。智能体应该在后台工作并浮现结果，而不是需要持续的保姆式照看
- **知识孤岛**：除非你手动桥接，否则营销研究的见解不会自动通知开发优先级

## 功能介绍

- **专业智能体**：每个智能体都有独特的角色、个性和针对其领域优化的模型
- **共享记忆**：项目文档、目标和关键决策对所有智能体均可访问——没有任何东西会丢失
- **私人上下文**：每个智能体还维护自己的对话历史记录和领域特定笔记
- **单一控制平面**：所有智能体都可以通过一个 Telegram 群聊访问——标记你需要的智能体
- **计划的每日任务**：智能体在未被询问的情况下主动工作——内容提示、竞争对手监控、指标追踪
- **并行执行**：多个智能体可以同时处理独立任务

## 示例团队配置

### 智能体 1：Milo（战略负责人）

```text
## SOUL.md — Milo

你是 Milo，团队负责人。自信、大局观、有魅力。

职责：
- 战略规划和优先级排序
- 协调其他智能体
- 每周目标设定和 OKR 追踪
- 将所有智能体的见解综合为可操作的决策

模型：Claude Opus
渠道：Telegram（响应 @milo）

每日任务：
- 上午 8:00：审查过夜智能体活动，发布早间站会摘要
- 下午 6:00：每日回顾，包括每周目标的进度
```

### 智能体 2：Josh（业务与增长）

```text
## SOUL.md — Josh

你是 Josh，业务分析师。务实、直奔主题、数据驱动。

职责：
- 定价策略和竞争分析
- 增长指标和 KPI 追踪
- 收入建模和单位经济学
- 客户反馈分析

模型：Claude Sonnet（快速、分析型）
渠道：Telegram（响应 @josh）

每日任务：
- 上午 9:00：提取并总结关键指标
- 每周追踪竞争对手定价变化
```

### 智能体 3：营销智能体

```text
## SOUL.md — 营销智能体

你是营销研究员。有创造力、好奇心、关注趋势。

职责：
- 内容构思和起草
- 竞争对手社交媒体监控
- 相关主题的 Reddit/HN/X 趋势追踪
- SEO 关键词研究

模型：Gemini（擅长网络研究和长上下文分析）
渠道：Telegram（响应 @marketing）

每日任务：
- 上午 10:00：根据热门话题浮现 3 个内容创意
- 每天监控竞争对手 Reddit/X 提及
- 每周内容日历草稿
```

### 智能体 4：开发智能体

```text
## SOUL.md — 开发智能体

你是开发智能体。精确、彻底、注重安全。

职责：
- 编码和架构决策
- 代码审查和质量检查
- 错误调查和修复
- 技术文档

模型：Claude Opus / Codex（用于实现）
渠道：Telegram（响应 @dev）

每日任务：
- 检查 CI/CD 流水线健康状况
- 审查未决 PR
- 标记技术债务项目
```

## 所需技能

- `telegram` 技能用于共享控制界面
- `sessions_spawn` / `sessions_send` 用于多智能体协调
- 共享文件系统或笔记工具用于团队记忆
- 不同模型提供商的单独 API 密钥（如果使用混合模型）
- 用于运行智能体的 VPS 或始终在线机器

## 如何设置

### 1. 共享记忆结构

```text
team/
├── GOALS.md           # 当前 OKR 和优先级（所有智能体读取）
├── DECISIONS.md       # 关键决策日志（仅追加）
├── PROJECT_STATUS.md  # 当前项目状态（由所有人更新）
├── agents/
│   ├── milo/          # Milo 的私人上下文和笔记
│   ├── josh/          # Josh 的私人上下文
│   ├── marketing/     # 营销智能体的研究
│   └── dev/           # 开发智能体的技术笔记
```

### 2. Telegram 路由

配置一个所有智能体都监听的 Telegram 群组，但每个智能体仅在被标记时响应：

```text
## AGENTS.md — Telegram 路由

Telegram 群组："Team"

路由：
- @milo → 战略智能体（孵化/恢复 milo 会话）
- @josh → 业务智能体（孵化/恢复 josh 会话）
- @marketing → 营销智能体（孵化/恢复 marketing 会话）
- @dev → 开发智能体（孵化/恢复 dev 会话）
- @all → 广播给所有智能体
- 无标记 → Milo（团队负责人）默认处理

每个智能体：
1. 读取共享 GOALS.md 和 PROJECT_STATUS.md 以获取上下文
2. 读取其自己的私人笔记
3. 处理消息
4. 在 Telegram 中响应
5. 如果响应涉及决策或状态更改，则更新共享文件
```

### 3. 计划任务

```text
## HEARTBEAT.md — 团队日程

每日：
- 上午 8:00：Milo 发布早间站会（聚合过夜智能体活动）
- 上午 9:00：Josh 提取关键指标
- 上午 10:00：营销根据热门话题浮现内容创意
- 下午 6:00：Milo 发布每日回顾

持续进行：
- 开发：监控 CI/CD 健康状况，审查进入的 PR
- 营销：Reddit/X 关键词监控（每 2 小时）
- Josh：竞争对手定价检查（每周）

每周：
- 周一：Milo 起草每周优先级（来自所有智能体的输入）
- 周五：Josh 编译每周指标报告
```

## 关键见解

- **个性比你想象的更重要**：给智能体起独特的名字和沟通风格，使得“与你的团队交谈”变得自然，而不是与通用的 AI 搏斗
- **共享记忆 + 私人上下文**：这种结合至关重要——智能体需要共同基础（目标、决策），但也需要自己的空间来积累领域专业知识
- **合适的工作用合适的模型**：不要使用昂贵的推理模型进行关键词监控。将模型能力与任务复杂性相匹配
- **计划任务是飞轮**：当智能体主动浮现见解，而不仅仅是在你询问时，真正的价值才会显现
- **从 2 个开始，而不是 4 个**：从一个负责人 + 一个专家开始，然后在发现瓶颈时添加智能体

## 灵感来源

此模式由 [Trebuh 在 X 上](https://x.com/iamtrebuh/status/2011260468975771862) 描述，他是一位独立创始人，设置了 4 个 OpenClaw 智能体——Milo（战略负责人）、Josh（业务）、一个营销智能体和一个开发智能体——所有这些都通过 VPS 上的单个 Telegram 聊天进行控制。每个智能体都有自己的个性、模型和计划任务，同时共享项目记忆。他将其描述为“一个全天候可用的真正的具体队”。

该模式也在 [OpenClaw Showcase](https://openclaw.ai/showcase) 上得到确认，其中 `@jdrhyne` 报告运行“15+ 智能体，3 台机器，1 个 Discord 服务器——IT 构建了其中的大部分，仅仅通过聊天”，`@nateliason` 描述了一个多模型流水线（原型 → 总结 → 优化 → 实现 → 重复），在每个阶段使用不同的模型。另一位用户 `@danpeguine` 运行两个不同的 OpenClaw 实例，在同一个 WhatsApp 群组中协作。
