# 市场研究与产品工厂

你想构建一个产品，但不知道构建什么。或者你有一家企业，需要了解你的客户正在为何苦苦挣扎。此工作流使用 Last 30 Days 技能从 Reddit 和 X 挖掘真实的痛点，然后让 OpenClaw 构建解决这些问题的方案。

## 功能介绍

- 使用 [Last 30 Days](https://github.com/matvanhorde/last-30-days) 技能研究过去 30 天内 Reddit 和 X 上的任何主题
- 浮现人们发布的真实挑战、抱怨和功能请求
- 帮助你从真正的用户痛点中识别产品机会
- 更进一步：要求 OpenClaw 构建一个解决其中一个挑战的 MVP
- 创建一个完整的从研究到产品的流水线，你无需编写任何代码

## 痛点

大多数有抱负的企业家都在为“构建什么”的问题而苦恼。传统的市场研究意味着要在论坛、社交媒体和评论网站上进行数小时的手动浏览。这自动化了整个从发现到原型的流水线。

## 所需技能

- Matt Van Horde 的 [Last 30 Days](https://github.com/matvanhorde/last-30-days) 技能
- 用于接收研究报告的 Telegram 或 Discord 集成

## 如何设置

1. 安装 Last 30 Days 技能：
```text
安装此技能：https://github.com/matvanhorde/last-30-days
```

2. 对任何主题运行研究：
```text
请使用 Last 30 Days 技能研究人们在 [你的主题] 方面遇到的挑战。

将发现组织为：
- 主要痛点（按频率排序）
- 具体抱怨和功能请求
- 现有解决方案的差距
- 新产品的机会
```

3. 选择一个痛点并让 OpenClaw 构建解决方案：
```text
为我构建一个解决 [研究中的痛点] 的 MVP。
保持简单——只有核心功能。
将其作为一个我可以与人分享的 Web 应用程序发布。
```

4. 对于持续的市场情报，安排定期研究：
```text
每周一早上，使用 Last 30 Days 技能研究人们在 Reddit 和 X 上关于 [你的利基市场] 说些什么。总结主要机会并将其发送到我的 Telegram。
```

## 真实世界示例

```text
"使用 Last 30 Days 技能研究人们在使用 OpenClaw 时遇到的挑战。"

结果：
- 设置困难：许多用户在初始配置上遇到困难
- 技能发现：人们找不到能满足他们需求的技能
- 成本问题：用户想要更便宜的模型替代品

→ "为我构建一个简单的 Web 应用程序，通过引导向导使 OpenClaw 设置更容易。"

OpenClaw 构建该应用程序。你发布它。你有了一个产品。
```

## 关键见解

- 这是 **自动驾驶式的创业**：发现问题 → 验证需求 → 构建解决方案，全部通过短信完成。
- Last 30 Days 技能给你真实、未经过滤的用户情绪——而不是经过消毒的调查数据。
- 你不需要懂技术。OpenClaw 既做研究又做构建。
- 安排每周研究以掌握市场上不断演变的痛点。
